Infrastructure-as-code can be more than a deployment mechanism. When done well, it becomes living documentation that explains how cloud systems are assembled, why they exist, and how they should evolve.

This session shows how to make IaC clearer, more expressive, and more maintainable so teams can treat infrastructure definitions as part of the system's documentation.
