API design gets harder when the consumers are no longer just humans. AI agents need stable, explicit contracts to interact safely and predictably with services, which makes contract-first thinking even more important.

This session explores how to design APIs that are easier to understand, easier to validate, and easier to evolve without surprising downstream consumers.
